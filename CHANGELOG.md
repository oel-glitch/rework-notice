# История изменений OATI PDF Parser

## [v4.2] - Ноябрь 2025

### 🎯 Основные изменения

#### Упрощенный EZP Workflow
- **Автоматическая загрузка шаблона**: Программа больше НЕ требует ручной загрузки Excel файла
- **Шаблон из памяти**: Автоматическое использование `templates/ezp_template.xls`
- **Уникальные файлы результатов**: Каждая обработка создает новый файл `output/Отправка_YYYYMMDD_HHMMSS.xlsx`
- **Исправлен интерфейс**: Удалена секция загрузки Excel, исправлено наложение элементов
- **Новое логирование**: Обновлены сообщения для отражения автоматического workflow

### 📋 Workflow изменения

**БЫЛО** (v4.1):
1. Загрузить шаблон Excel вручную
2. Выбрать PDF файлы
3. Нажать "Обработать"
4. Результат перезаписывает исходный файл

**СТАЛО** (v4.2):
1. Бросить PDF файлы
2. Нажать "Обработать ЭЗП"
3. Готово! → `output/Отправка_YYYYMMDD_HHMMSS.xlsx`

### 📚 Новая документация

- **YANDEX_API_SETUP.txt**: Подробная инструкция по получению YandexGPT API ключей
  - Пошаговое руководство с скриншотами описанием интерфейса
  - Настройка через Windows Credential Manager
  - Решение типичных проблем
  - Информация о тарификации
  
- **INSTRUCTION_RU.txt**: Добавлены разделы:
  - Раздел 9: Модуль ЭЗП с упрощенным workflow
  - Раздел 10: AI Помощник и настройка
  - Обновлена версия до 4.2

### 🔧 Технические изменения

#### src/ezp_processor.py
- `process_ezp()` больше не требует параметр `excel_path`
- Новый параметр `output_filename` для кастомизации имени файла
- Автоматическое копирование `templates/ezp_template.xls` в `output/`
- Генерация уникальных имен с timestamp
- Обновлена документация метода

#### main.py
- Удалена секция UI для загрузки Excel (строки ~594-612)
- Исправлен grid layout во вкладке ЭЗП (row 0, 1, 2)
- Удалены функции `load_excel_file()` и `update_ezp_info()`
- Обновлен `process_ezp_batch()` для работы с новым API
- Legacy функции `process_ezp()` и `process_ezp_thread()` перенаправляют на новый workflow

### ✅ Преимущества

- **На 2 шага меньше** в пользовательском workflow
- **Нет риска** перезаписать важный файл
- **Уникальные имена** для каждой обработки
- **Чистый интерфейс** без лишних элементов
- **Автоматическое управление** файлами

### ⚠️ Важные замечания

- Шаблон `templates/ezp_template.xls` должен существовать
- Результаты сохраняются ТОЛЬКО в папку `output/`
- Каждая обработка создает новый файл с timestamp
- Старые результаты НЕ перезаписываются

## [v4.1] - Ноябрь 2025

### 🆕 Новые функции

#### AI Помощник (YandexGPT Integration)
- **YandexGPT AI для генерации текста**: Интеграция с YandexGPT API для автоматической генерации официальных параграфов в уведомлениях
- **AI-assisted генерация union_paragraph**: AI генерирует текст про рассмотрение вопроса Объединением на основе вопроса из резолюции
- **Graceful fallback**: При отсутствии AI или проблемах с подключением - автоматический возврат к rule-based генерации
- **UI таб "🤖 AI Помощник"**: Статус подключения (🟢/🔴), тестирование соединения, управление API credentials
- **Secure credentials storage**: API Key и Folder ID хранятся в Windows Credential Manager
- **Offline-first архитектура**: AI - опциональное улучшение, приложение работает полностью без него
- **Конфигурация через JSON**: Все prompts и настройки AI в `config/ai_settings.json`

#### MOSEDO Автоматизация
- **Автоматическая запись workflow через браузер**: Теперь можно записать последовательность действий в MOSEDO прямо в браузере, не вводя шаги вручную
- Новые кнопки в интерфейсе: "🔴 Записать мой workflow" и "⏹ Остановить запись"
- Использует Selenium для автоматического захвата действий пользователя
- Сохранение записанных workflow в базу данных для последующего воспроизведения

#### Система версионирования
- **Автоматическое отслеживание версий**: Каждая сборка теперь содержит уникальный git hash и время сборки
- **Отображение версии в приложении**: Версия с git hash видна в заголовке окна и логах запуска
- **Версионирование архивов**: ZIP архивы включают git hash в имени файла для точной идентификации
- Файл `version.txt` внедряется в .exe при сборке для проверки актуальности

#### Scanner Integration (Canon DR-M260)
- Установлен Tesseract OCR 5.5.0 для распознавания текста
- Готовность к интеграции со сканером через TWAIN/WIA драйверы
- Поддержка пакетного сканирования и создания searchable PDF

### 🐛 Исправления ошибок

#### Форматирование организаций в уведомлениях
- **Исправлено**: Множественные организации теперь правильно форматируются с использованием русской грамматики
- **Было**: "Департамент А, Департамент Б"
- **Стало**: "в Департамент А и в Департамент Б"
- Корректная обработка для 1, 2 и 3+ организаций

#### Russia Post API Integration
- **Исправлено**: Ошибка NoneType при неудачной инициализации API
- **Исправлено**: Невозможность повторной попытки подключения после исправления учетных данных
- Graceful degradation при отсутствии credentials или сетевых проблемах
- Корректная обработка offline режима с информативными сообщениями
- Возможность retry после исправления конфигурации без перезапуска приложения

### 🔧 Технические улучшения

- Новый модуль `src/version_info.py` для централизованного управления версиями
- Обновленный `build.py` с автоматической генерацией `version.txt`
- Улучшенная обработка ошибок в Russia Post API с детальной диагностикой
- Добавлен CHANGELOG.md для отслеживания изменений между версиями

### 📝 Документация

- Создан CHANGELOG.md с историей изменений
- Обновлен replit.md с информацией о MOSEDO workflow recording
- Добавлена информация о системе версионирования

## Как проверить версию вашего приложения

### В запущенном приложении:
1. Посмотрите на заголовок окна - там указана версия и git hash
2. Пример: `Парсер PDF и генератор Word документов - ОАТИ - v4.1 (abc1234)`

### В консоли при запуске:
```
======================================================================
ЗАПУСК ПРИЛОЖЕНИЯ PDF Parser ОАТИ v4.1 (abc1234) - 2025-11-18 12:00:00
======================================================================
```

### В имени ZIP архива:
- Формат: `OATI_PDF_Parser_v4.1_{git_hash}_{дата}.zip`
- Пример: `OATI_PDF_Parser_v4.1_abc1234_20251118.zip`
